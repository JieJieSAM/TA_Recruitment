# Handout Analysis (EBU6304)

## Mandatory Constraints

- Must be Java application:
  - stand-alone Java app, or
  - lightweight Java Servlet/JSP web app
- Must use plain file storage (JSON/CSV/XML/TXT)
- Must NOT use a database

## Suggested Stakeholders

- Teaching Assistant (TA)
- Module Organiser (MO)
- Admin

## Suggested Functional Candidates

- TA profile creation
- CV upload
- Job discovery
- Job application
- Application status tracking
- MO job posting
- MO applicant selection
- Admin workload monitoring

## AI-Related Notes

- AI is encouraged but must be used critically
- Outputs should be explainable
- Prefer hybrid design:
  - deterministic rules for core decisions
  - optional AI-generated explanation/recommendation layer

## Assessment Implications

- First assessment focuses on backlog quality, prototype usability, and evidence
- Intermediate checks iterative progress and team collaboration on GitHub
- Final checks design, implementation, testing, and acceptance demonstration

## Initial Scope Recommendation

Core workflow first:
1. MO posts job
2. TA applies for job
3. MO reviews and updates status
4. TA tracks status
5. Admin views workload summary
